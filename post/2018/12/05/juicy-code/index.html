Algorithm Review Memorization

Heap

  1. Build Heap —- O(n)

Disjoint Set

  1. linked-list representation
    • make —- O(1)
    • find —- O(1)
    • union amortized time —- O(n) or O(m + nlgn) with the weighted union heuristic, where m is for the makes and finds
  2. rooted-tree representation
    • union = two finds and one link
    • make —- O(1)
    • link —- O(1)
    • find
      1. rank heuristic —- O(lgn)
      2. path compression —- shit
      3. both —- $O(m\alpha(n))$

Hash

Collision solution: 1. Chaining * insert —- O(1) * delete —- O(1), the pointer of the element is given, so no search is needed! * search * $\alpha$ = n/m, average number of elements per linked list * Unsuccessful —- O(1+$\alpha$) * Successful —- O(1+$\alpha$) * Worse —- O(n) 2. Open addressing
